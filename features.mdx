## Document Creation and Editing

Create new documents directly from the dashboard. You access a full-featured editor supporting Markdown, JSX components, and embeds.

<Steps>
  <Step title="Create Document" icon="plus">
    Click the `New Document` button in your workspace.
  </Step>
  <Step title="Edit Content" icon="edit">
    Use the visual editor or switch to Markdown mode.
  </Step>
  <Step title="Add Components" icon="layers">
    Insert `<Callout>`, `<Tabs>`, or code blocks via the toolbar.
  </Step>
  <Step title="Preview & Publish" icon="eye">
    Preview changes and publish to share.
  </Step>
</Steps>

For advanced editing, integrate custom scripts.

<CodeGroup tabs="JavaScript,Python">
  ```javascript
  // Add custom component in MDX
  import { Callout } from './components';

  <Callout kind="info">
    This is a custom note.
  </Callout>
  ```
  ```python
  # Embed Python snippets in docs
  def create_doc(title):
      return f"# {title}\n\nAdd your content here."
  ```
</CodeGroup>

<Callout kind="tip">
  Enable real-time preview to see changes as you type.
</Callout>

## Folder and Page Organization

Organize your documentation hierarchically with folders and pages. You nest content to match your project's structure.

<Tabs>
  <Tab title="Folders" icon="folder">
    Create folders to group related pages.

    ```
    docs/
    ├── api/
    │   ├── endpoints.mdx
    │   └── auth.mdx
    └── guides/
        └── quickstart.mdx
    ```
  </Tab>
  <Tab title="Page Linking" icon="link">
    Link pages using relative paths: `[Quickstart](./quickstart.mdx)`.
  </Tab>
  <Tab title="Custom Navigation" icon="menu">
    Configure sidebar menus in `nav.json`.
  </Tab>
</Tabs>

## Search and Version Control

Tyler Durden's search indexes all content for instant results. Filter by title, content, or tags.

| Feature | Description |
|---------|-------------|
| Full-Text Search | Matches keywords across pages |
| Filters | By folder, tag, or date |
| History | View page versions |

Track changes with built-in version control.

<Expandable title="Advanced Version Control" default-open="false">
  Revert to previous versions or compare diffs.

  ```bash
  # CLI command to checkout version
  tyler version checkout v1.2.0 --page=features.mdx
  ```

  Merge conflicts resolve automatically in most cases.
</Expandable>

<Callout kind="info">
  Search updates in real time as you add content.
</Callout>
